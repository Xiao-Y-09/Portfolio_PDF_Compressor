# 数据契约

> **地位**：本文档是项目三份真源文档之一，是 `backend/app/contracts/`（构建 Phase 3）的唯一实现依据。
>
> **铁律 6**：Phase N 输出的数据结构一旦定义，Phase N+1 只能消费不能修改。任何下游 Phase 发现契约不够用，必须回到本文档 + `contracts/` 重新定义，不得私自加字段。
>
> **实现铁律**：全部使用 Pydantic BaseModel，禁止使用裸 dict 或 dataclass。

### 1.6 data_ref 路径规范（决策记录：2026-07-04）

所有 `*_ref` 字段（`page_bytes_ref`、`data_ref`、`path_data_ref`、`subsetted_data_ref`、
`compressed_data_ref`）统一为：**相对 `PipelineContext.tmp_workspace` 的 POSIX 风格相对路径**
（如 `images/img_001_002.bin`，分隔符一律 `/`）。

- **唯一解析入口**：任何 Phase 读写 ref 指向的文件，必须经 `PipelineContext.resolve_ref(ref)`；
  禁止手工字符串拼接路径。
- **resolve_ref 安全校验**（防路径穿越）：拒绝绝对路径、盘符、`..` 分段、空串、`\` 分隔符；
  解析结果必须仍位于 tmp_workspace 内，否则 ValueError。
- **备选方案的取舍**：绝对路径（否——workspace 迁移/session 恢复即失效，且泄露服务器布局）；
  纯标识符（否——需要额外的映射解析器，本质是换写法的相对路径；防穿越保证由唯一解析
  卡口提供，与标识符方案等价，且相对路径保留人类可调试性，并与手册 Phase 4/5 原文一致）。

### 1.5 全局验证约束（契约层强制，测试必须覆盖）

- `quality ∈ [0, 100]`（决策层再收紧到 [quality_floor, quality_ceiling]；
  Tier 3 收紧到 [tier3_quality_floor, quality_ceiling]——config 相关界一律
  决策层负责，契约层只管通用界，避免契约耦合 config）
- `target_dpi > 0`
- `area_ratio ∈ [0, 1]`
- PagePlan 互斥约束由 model_validator 强制（whole_page_plan/plans 互斥、
  skip 页全空，见第 6 章，修订 2026-07-04）
- 所有类型可 JSON round-trip：`model_validate(model_dump())` 恒等

### 3.1 元素类型（elements.py）

```python
RasterImage {
    data_ref: str           # 文件内引用（如 images/img_{page}_{index}.bin），不直接存 bytes
    width: int              # 原始像素宽
    height: int             # 原始像素高
    dpi: float              # 反算 DPI = width / (bbox.w / 72)
    color_space: str        # RGB / CMYK / Gray ...
    has_alpha: bool         # = (alpha_type == "translucent")，兼容性快捷字段
    alpha_type: "none" | "opaque" | "translucent"
                            # 修订 2026-07-04（第 2 批）：
                            #   none        = 无 alpha 通道
                            #   opaque      = 有 SMask 但 ≥99% 像素为 255（伪透明）
                            #   translucent = 真透明
                            # Producer = Phase 5 提取时对 SMask 像素采样统计（数百点，不扫全图）。
    is_grayscale: bool      # 修订 2026-07-04（质量优化 2）：默认 False。
                            # Producer = Phase 5 像素采样（缩略后逐点 |R-G|≤2 且 |R-B|≤2
                            # 比例 >95%，容差 2 吸收 JPEG 色度噪声；L 模式直接 True）。
                            # Consumer = Phase 9 编码前转 L/LA 模式（3→1 通道）。
    original_bytes: int     # 原始图片字节数（≥0）。修订 2026-07-04：Producer = Phase 5
                            # 提取写 data_ref 文件时记录 len(bytes)；Consumer = Phase 8
                            # 预算页权重（bytes × area_ratio）与 RasterPlan.original_bytes。
                            # 断链原因：decide 是纯函数不能 stat 文件，此前该值无来源。
    bbox: BBox
}

VectorPath {
    path_data_ref: str          # pickle 序列化路径数据的引用：vectors/vec_{page}_{index}.bin
    control_point_count: int    # 所有路径命令控制点总数
    stroke_color: Optional[str]
    fill_color: Optional[str]
    original_bytes: int         # 【语义修订 2026-07-04 第 2 次】"PDF 内容流预估输出字节数"
                                # = control_point_count × vector_bytes_per_control_point
                                #  （config 键，默认 3.0，Phase 10 首次产出真实 PDF 后校准）。
                                # Producer = Phase 5；Consumer = Phase 7 固定开销 /
                                # Phase 8 VectorPlan.original_bytes 与体积估算。
                                # ⚠ 不是 pickle 文件大小：pickle 实测 ~31.6 B/控制点，
                                # 而 PDF 内容流仅 ~2-4 B/控制点（数字文本+deflate）——
                                # Producer-Consumer 审计发现的单位不匹配（Consumer 要
                                # 输出成本，pickle 是内存序列化成本，膨胀 10 倍导致
                                # total_fixed_bytes 205MB > 104MB 源文件）。
                                # pickle 文件照旧存盘（Phase 10 重绘用），仅不作为度量。
    bbox: BBox                  # 所有路径的最小外接矩形
}

TextBlock {
    content: str
    font_ref: str           # 字体名（对应 fonts_used 的 key）
    font_size: float
    bbox: BBox
}

    ref_id: str
    xobject_type: Literal["image", "form"]

}

    bbox: BBox
}
```

### 3.1.1 共享 data_ref 设计（决策记录 2026-07-04，用户批准）

**data_ref 不保证唯一**：同一源图像（同一 PDF xref）在页面上被多处摆放时，
Phase 5 每页只写盘一次字节，所有摆放条目**共享同一个 data_ref**（bbox/dpi 各自不同）。

**为什么**：真实样本实测单页 6000+ 摆放 vs 全文档仅 ~300 个唯一图像（贴图被平铺
数千次）。若逐摆放独立存储/压缩/嵌入，Phase 9 会把同一张图压几千遍（CPU 爆炸），
Phase 10 会嵌入几千份压缩流（**输出比输入还大**——源 PDF 本来就只存一份）。

**Phase 9/10 的配合要求（实现时必须遵守）**：
- Phase 9（compress.py）：按 data_ref 分组，**每个唯一 data_ref 只压缩一次**
  （参数取该组所有摆放中最苛刻的需求，如最大 target_dpi），
  同组所有 ImageResult 引用同一 compressed_data_ref；
- Phase 10（assemble.py）：同一 compressed_data_ref 用 PyMuPDF 的 xref 共享插入
  （首次 insert_image 记下返回的 xref，后续摆放复用该 xref），不得重复嵌入字节流。

## 第 5 章：PreprocessResult（Phase 7 确定性预处理输出）

```python
SubsetResult {                  # fonts.py
    font_name: str
    original_bytes: int
    subsetted_bytes: int        # 无法子集化时 = original_bytes
    subsetted_data_ref: str     # fonts_subsetted/font_{name}.ttf
}

PerPageFixedBytes {             # preprocess.py
    page_number: int
    vector_bytes: int           # 该页矢量序列化数据大小之和
    text_bytes: int             # Σ len(content.utf8) × 1.5（含结构开销）
    annotation_bytes: int       # 每个注释估 100-500 字节
}

PreprocessResult {
    subsetted_fonts: List[SubsetResult]
    total_fixed_bytes: int      # = Σ per_page + Σ subsetted_bytes
                                #   + 元数据估算（清理后 500-2000 字节）
                                #   + PDF 结构开销（页数 × 200 + 500 字节）
    per_page_fixed_bytes: List[PerPageFixedBytes]
    stripped_metadata_fields: List[str]   # 默认清理 author, creator；保留 title
}
```

- 估算误差 ±10% 可接受，由 Phase 8/9 收敛循环兜住
- `total_fixed_bytes` 是 Phase 8 预算公式的直接输入：`raster_budget = target - total_fixed_bytes`

## 第 6 章：CompressionPlan（Phase 8 决策输出）

```python
RasterPlan {                    # plan.py
    image_index: int            # 对应 PageElements.raster_images 下标
    area_ratio: float           # ∈ [0, 1]
    original_dpi: float
    original_bytes: int
    target_dpi: int             # > 0，且 ≤ original_dpi（只降不升）

    max_dimension: int          # 最长边像素上限（像素天花板兜底）
    output_format: Literal["jpeg", "png"]

}

VectorPlan {
    path_group_index: int       # 对应 PageElements.vector_paths 下标
    area_ratio: float
    original_bytes: int
    strategy: Literal["keep", "simplify", "rasterize"]
    rasterize_dpi: Optional[int] = None        # 仅 strategy="rasterize"
    simplify_tolerance: Optional[float] = None # 仅 strategy="simplify"
}

CompressionTier = Literal["in_place", "hybrid", "full_raster"]
                                # 修订 2026-07-04（Tier 系统，《压缩决策引擎.md》§8.5）：
                                # 语义命名（用户裁决 3），不用 1/2/3 裸数字。
                                # 顺序即降级方向：in_place = 最保真，full_raster = 最极限。
                                # 索引可比较：判断"当前 Tier 是否已至最深"等用
                                # COMPRESSION_TIERS.index(tier)，不做硬编码字符串比较。
                                # COMPRESSION_TIERS 常量（tuple）与本 Literal 同源同序，
                                # 契约层导出，测试断言两者同步。

WholePageRasterPlan {           # 修订 2026-07-04（Tier 系统，§8.6）：整页光栅化计划
    dpi: int                    # Field(gt=0)。契约层为通用界；Tier 相关界
                                # （Tier 2: [dpi_floor, dpi_ceiling] / Tier 3:
                                # [tier3_dpi_floor, tier3_dpi_ceiling]）依赖 config 值，
                                # 按 §1.5 既有约定由决策层收紧 + 《压缩决策引擎.md》
                                # 第 10 章清单断言——契约层不耦合 config（与
                                # RasterPlan.quality 的处理方式完全一致）
    quality: int                # Field(ge=0, le=100)，Tier 界同上
    estimated_bytes: int        # Field(ge=0)，预算口径估算
}

PagePlan {
    page_number: int
    skip: bool                  # true → 所有 plan 字段为空/None
    raster_plans: List[RasterPlan]
    vector_plans: List[VectorPlan]
    whole_page_plan: Optional[WholePageRasterPlan] = None
                                # 修订 2026-07-04（Tier 系统）：置位 → 本页整页光栅化。
                                # Producer = decide（Tier 2 选页 / Tier 3 全选）；
                                # Consumer = Phase 9 渲染整页 JPEG + Phase 10 替换页内容。
}
# 契约层强约束（Pydantic model_validator，构造即校验，不靠下游代码自觉）：
#   ① whole_page_plan ≠ None 时 raster_plans == [] 且 vector_plans == []
#      （全页由单计划覆盖，混用即 ValidationError）
#   ② skip == True 时 whole_page_plan 必须为 None
#      （skip 页在任何 Tier 下都不被触碰，铁律 4 在契约层的落点）

CompressionPlan {

    pages: List[PagePlan]
    round_number: int           # 第几轮收敛循环，从 1 开始
    base_aggressiveness: float  # ∈ [0,1]，本轮决策使用的全局激进度。修订 2026-07-04：

                                # 至 CompressionResult → 下一轮 decide 的 prev_aggressiveness。
                                # 断链原因：第 N 轮调整公式需要上一轮激进度，仅凭
                                # previous_result 无法重建（需要完整 gap 历史才可推导）。
    tier: CompressionTier = "in_place"
                                # 修订 2026-07-04（Tier 系统）：本 Plan 所属层级。
                                # Producer = orchestrator 传入 decide；Consumer = Phase 9
```

## 第 7 章：CompressionResult（Phase 9 执行输出）

```python
ImageResult {                   # result.py
    page_number: int
    image_index: int
    compressed_bytes: int       # 实际大小（压缩失败用原图凑数时 = original_bytes）
    compressed_data_ref: str    # compressed/img_{page}_{index}.{ext}
    actual_dpi: float           # 实际执行到的 DPI
    actual_quality: int         # 实际使用的 quality
}

CompressionResult {
    total_raster_bytes: int     # Σ compressed_bytes
    per_image_results: List[ImageResult]
    round_number: int
    base_aggressiveness: float  # ∈ [0,1]。修订 2026-07-04：Phase 9 从 CompressionPlan
                                # 原样回带（不做任何计算），供下一轮 decide 读取为
                                # prev_aggressiveness（收敛反馈环的闭合点）。
    tier_used: CompressionTier = "in_place"
                                # 修订 2026-07-04（Tier 系统）：Phase 9 从
                                # CompressionPlan.tier 原样回带（同 base_aggressiveness
                                # 通路）。Consumer = orchestrator（任务返回 dict 的
                                # tier_used/warning）→ Phase 12 API 透传 → Phase 14
                                # 前端警告展示（look-ahead 预纳入）。
}
```

- 消费者有二：Orchestrator（判断 gap 是否达标）+ 下一轮 Phase 8（`previous_result` 反馈调整激进度）
- 任务层输出（非契约模型，dict）：`tier_used ≠ "in_place"` 时 resume 任务返回 dict
  增加 `warning` 字段（文案见《压缩决策引擎.md》§8.5），Phase 12 REST 层原样透传

### 字段来源审计原则（Phase 3 完成后的关键检查）

把所有类型的字段画成 DAG：每个字段要么来自上一个 Phase 的输出，要么是该 Phase 新引入（有明确来源：PDF 解析 / 用户输入 / config / 计算得出）。发现"凭空出现"或"来源不明"的字段 → 契约设计有漏洞，回本文档补齐。

| 示例字段 | 来源 |
|---------|------|
| `RasterPlan.area_ratio` | 计算：`RasterImage.bbox.area / (PageElements.page_width × page_height)` |
| `RasterPlan.original_dpi` | `RasterImage.dpi`（Phase 5 反算） |
| `CompressionPlan.raster_budget` | `UserPreferences.target_size_mb` − `PreprocessResult.total_fixed_bytes` |
| `ImageResult.image_index` | `RasterPlan.image_index` ← `raster_images` 下标 |
| `PagePlan.skip` | `UserPreferences.per_page_overrides[].action == "skip"` |
| `RasterImage.original_bytes` | Phase 5 写 data_ref 文件时记录 `len(bytes)` → Phase 8 页权重与 `RasterPlan.original_bytes` |
| `VectorPath.original_bytes` | Phase 5 写 path_data_ref 文件时记录 → Phase 8 `VectorPlan.original_bytes` / 矢量估算基数 |
| `CompressionResult.base_aggressiveness` | Phase 8 记入 Plan → Phase 9 原样回带 → 下一轮 Phase 8 的 prev_aggressiveness |
| `PipelineContext.convergence_history` | Phase 9 每轮追加 → Phase 11 日志诊断 / Phase 13 收敛统计 / Phase 14 可选图表 |
| `CompressionPlan.tier` | orchestrator 降级状态机传入 decide → Phase 9 判定整页计划 → `tier_used` 回带 |
| `PagePlan.whole_page_plan` | decide Tier 2 选页纯函数 / Tier 3 全选 → Phase 9 渲染 + Phase 10 页内容替换 |
| `CompressionResult.tier_used` | `CompressionPlan.tier` 原样回带 → 任务返回 dict → Phase 12 API → Phase 14 前端 |
| `PhaseErrorData.diagnostics` | orchestrator Tier 3 失败时构造（skip 统计 + 末轮真实 assemble 大小）→ API 错误响应 → 前端可行动建议 |
