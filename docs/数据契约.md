# 数据契约

> **地位**：本文档是项目三份真源文档之一，是 `backend/app/contracts/`（构建 Phase 3）的唯一实现依据。
>
> **铁律 6**：Phase N 输出的数据结构一旦定义，Phase N+1 只能消费不能修改。任何下游 Phase 发现契约不够用，必须回到本文档 + `contracts/` 重新定义，不得私自加字段。
>
> **实现铁律**：全部使用 Pydantic BaseModel，禁止使用裸 dict 或 dataclass。

---

## 第 1 章：类型定义总览

### 1.1 契约文件与类型分布

| 文件 | 类型 | 属于哪个 Phase 的输出 |
|------|------|---------------------|
| `contracts/geometry.py` | BBox | 基础类型（全 Phase 共用） |
| `contracts/elements.py` | RasterImage, VectorPath, TextBlock, XObject, Annotation | Phase 5（元素级） |
| `contracts/pages.py` | RawPage, PageElements, ClassifiedPage | Phase 4 / 5 / 6 |
| `contracts/fonts.py` | FontUsage, SubsetResult | Phase 5 / 7 |
| `contracts/preferences.py` | CompressionTarget, PageOverride, UserPreferences | 用户输入（API 层） |
| `contracts/plan.py` | RasterPlan, VectorPlan, PagePlan, CompressionPlan | Phase 8 |
| `contracts/result.py` | ImageResult, CompressionResult | Phase 9 |
| `contracts/preprocess.py` | PerPageFixedBytes, PreprocessResult | Phase 7 |
| `contracts/pipeline.py` | PipelineContext, PhaseError | 横切类型（运行时上下文/错误） |

### 1.2 基础几何类型：BBox（geometry.py）

```python
BBox {
    x: float        # 左下角 x（PDF 用户单位，point = 1/72 英寸）
    y: float
    w: float        # 宽（points）
    h: float        # 高（points）
}
# property area -> float                      # w × h
# method area_ratio(page_area: float) -> float  # area / page_area，∈ [0, 1]
```

### 1.3 用户偏好类型（preferences.py）

```python
CompressionTarget = Literal["screen", "print", "email"]

PageOverride {
    page_number: int
    action: Literal["skip", "aggressive", "reclassify"]
    new_type: Optional[str] = None      # action="reclassify" 时使用
}

UserPreferences {
    target_size_mb: float               # 5 / 10 / 15 / 20
    compression_target: CompressionTarget
    per_page_overrides: List[PageOverride] = []
}
```

### 1.4 横切类型（pipeline.py）

```python
PipelineContext {
    file_id: str            # 上传文件的 UUID v4
    tmp_workspace: str      # 本次任务的临时工作区根目录
}
# 携带所有 Phase 共享的运行时上下文。
# 注意：data_ref / page_bytes_ref 等引用均为相对 tmp_workspace 的路径。

PhaseError {
    phase: str              # "split" / "extract" / "decide" / "orchestrator" / "assemble" ...
    code: str               # "INVALID_PDF" / "ENCRYPTED_PDF" / "TARGET_TOO_SMALL" /
                            # "CONVERGENCE_FAILED" / "ASSEMBLY_FAILED" ...
    message: str
    recoverable: bool
}
```

### 1.5 全局验证约束（契约层强制，测试必须覆盖）

- `quality ∈ [0, 100]`（决策层再收紧到 [quality_floor, quality_ceiling]）
- `target_dpi > 0`
- `area_ratio ∈ [0, 1]`
- 所有类型可 JSON round-trip：`model_validate(model_dump())` 恒等

## 第 2 章：RawPage（Phase 4 拆页输出）

```python
RawPage {
    page_number: int        # 从 1 开始（对齐 PDF 传统）
    page_width: float       # PDF 用户单位（points）
    page_height: float
    page_bytes_ref: str     # 单页 PDF 字节流的相对路径：pages/page_{N}.pdf
}
```

- 生产者：`pipeline/split.py`；消费者：Phase 5
- 语义：每页的原始字节流保存到 `tmp_workspace/pages/`，契约里只存引用不存 bytes
- 单页读取失败允许缺页（数量可能少于原始页数，缺页记 warning）

## 第 3 章：PageElements + fonts_used + metadata（Phase 5 提取输出）

### 3.1 元素类型（elements.py）

```python
RasterImage {
    data_ref: str           # 文件内引用（如 images/img_{page}_{index}.bin），不直接存 bytes
    width: int              # 原始像素宽
    height: int             # 原始像素高
    dpi: float              # 反算 DPI = width / (bbox.w / 72)
    color_space: str        # RGB / CMYK / Gray ...
    has_alpha: bool         # SMask 存在 → True（注意检测伪透明）
    bbox: BBox
}

VectorPath {
    path_data_ref: str          # pickle 序列化路径数据的引用：vectors/vec_{page}_{index}.bin
    control_point_count: int    # 所有路径命令控制点总数
    stroke_color: Optional[str]
    fill_color: Optional[str]
    bbox: BBox                  # 所有路径的最小外接矩形
}

TextBlock {
    content: str
    font_ref: str           # 字体名（对应 fonts_used 的 key）
    font_size: float
    bbox: BBox
}

XObject {
    ref_id: str
    xobject_type: Literal["image", "form"]
    bbox: BBox
}

Annotation {
    anno_type: Literal["link", "bookmark", "comment", "form_field"]
    target: str             # link → uri；bookmark → 目标页等
    bbox: BBox
}
```

### 3.2 页级聚合（pages.py）

```python
PageElements {
    page_number: int
    page_width: float
    page_height: float
    raster_images: List[RasterImage]
    vector_paths: List[VectorPath]
    text_blocks: List[TextBlock]
    xobjects: List[XObject]
    annotations: List[Annotation]
}
```

单页提取失败 → 该页返回空 PageElements（各列表为空），不阻塞后续。

### 3.3 全局字体表（fonts.py）

```python
FontUsage {
    font_name: str
    data_ref: str               # fonts/font_{name}.ttf；提取失败时为 None（Phase 7 跳过子集化）
    chars_used: Set[str]        # 跨页累计的使用字符集（文字块提取阶段累加）
}
# Phase 5 返回 Dict[str, FontUsage]，key = font_name
```

### 3.4 全局元数据

```python
metadata: Dict[str, Any]
# doc.metadata：title, author, creator, producer, creation_date
# 注：这是 Phase 5 三元组返回值之一，属于原始提取物，
#     清理决策在 Phase 7（见 PreprocessResult.stripped_metadata_fields）
```

Phase 5 完整返回签名：

```python
extract_elements(...) -> Tuple[List[PageElements], Dict[str, FontUsage], Dict[str, Any]]
```

## 第 4 章：ClassifiedPage（Phase 6 分类输出）

```python
ClassifiedPage {
    page_number: int
    page_type: Literal["photo_heavy", "text_heavy", "chart", "mixed"]
    dominant_element: Literal["raster", "vector", "text"]
    raster_area_ratio: float    # ∈ [0, 1]
    vector_area_ratio: float
    text_area_ratio: float
    complexity_score: float     # ∈ [0, 1]，分类边界模糊度：0=非常明确，1=非常模糊
                                # 高分 → 前端提示"这一页分类不太确定，请手动确认"
}
```

- 生产者：`pipeline/classify.py`（V1 启发式）；消费者：用户 review 界面 + Phase 8
- V2 AI 分类器可替换实现，但输入输出契约（`List[PageElements] → List[ClassifiedPage]`）绝对不变
- 用户 review 修改后回传的仍是本类型（reclassify 改 page_type）

## 第 5 章：PreprocessResult（Phase 7 确定性预处理输出）

```python
SubsetResult {                  # fonts.py
    font_name: str
    original_bytes: int
    subsetted_bytes: int        # 无法子集化时 = original_bytes
    subsetted_data_ref: str     # fonts_subsetted/font_{name}.ttf
}

PerPageFixedBytes {             # preprocess.py
    page_number: int
    vector_bytes: int           # 该页矢量序列化数据大小之和
    text_bytes: int             # Σ len(content.utf8) × 1.5（含结构开销）
    annotation_bytes: int       # 每个注释估 100-500 字节
}

PreprocessResult {
    subsetted_fonts: List[SubsetResult]
    total_fixed_bytes: int      # = Σ per_page + Σ subsetted_bytes
                                #   + 元数据估算（清理后 500-2000 字节）
                                #   + PDF 结构开销（页数 × 200 + 500 字节）
    per_page_fixed_bytes: List[PerPageFixedBytes]
    stripped_metadata_fields: List[str]   # 默认清理 author, creator；保留 title
}
```

- 估算误差 ±10% 可接受，由 Phase 8/9 收敛循环兜住
- `total_fixed_bytes` 是 Phase 8 预算公式的直接输入：`raster_budget = target - total_fixed_bytes`

## 第 6 章：CompressionPlan（Phase 8 决策输出）

```python
RasterPlan {                    # plan.py
    image_index: int            # 对应 PageElements.raster_images 下标
    area_ratio: float           # ∈ [0, 1]
    original_dpi: float
    original_bytes: int
    target_dpi: int             # > 0，且 ≤ original_dpi（只降不升）
    quality: int                # ∈ [0, 100]（决策层收紧到 [30, 95]）
    max_dimension: int          # 最长边像素上限（像素天花板兜底）
    output_format: Literal["jpeg", "png"]
    estimated_bytes: int
}

VectorPlan {
    path_group_index: int       # 对应 PageElements.vector_paths 下标
    area_ratio: float
    original_bytes: int
    strategy: Literal["keep", "simplify", "rasterize"]
    rasterize_dpi: Optional[int] = None        # 仅 strategy="rasterize"
    simplify_tolerance: Optional[float] = None # 仅 strategy="simplify"
}

PagePlan {
    page_number: int
    skip: bool                  # true → raster_plans 与 vector_plans 为空
    raster_plans: List[RasterPlan]
    vector_plans: List[VectorPlan]
}

CompressionPlan {
    raster_budget: int          # = target_bytes - total_fixed_bytes
    pages: List[PagePlan]
    round_number: int           # 第几轮收敛循环，从 1 开始
}
```

语义约束见《压缩决策引擎.md》第 3 章输出约束。

## 第 7 章：CompressionResult（Phase 9 执行输出）

```python
ImageResult {                   # result.py
    page_number: int
    image_index: int
    compressed_bytes: int       # 实际大小（压缩失败用原图凑数时 = original_bytes）
    compressed_data_ref: str    # compressed/img_{page}_{index}.{ext}
    actual_dpi: float           # 实际执行到的 DPI
    actual_quality: int         # 实际使用的 quality
}

CompressionResult {
    total_raster_bytes: int     # Σ compressed_bytes
    per_image_results: List[ImageResult]
    round_number: int
}
```

- 消费者有二：Orchestrator（判断 gap 是否达标）+ 下一轮 Phase 8（`previous_result` 反馈调整激进度）

### 字段来源审计原则（Phase 3 完成后的关键检查）

把所有类型的字段画成 DAG：每个字段要么来自上一个 Phase 的输出，要么是该 Phase 新引入（有明确来源：PDF 解析 / 用户输入 / config / 计算得出）。发现"凭空出现"或"来源不明"的字段 → 契约设计有漏洞，回本文档补齐。

| 示例字段 | 来源 |
|---------|------|
| `RasterPlan.area_ratio` | 计算：`RasterImage.bbox.area / (PageElements.page_width × page_height)` |
| `RasterPlan.original_dpi` | `RasterImage.dpi`（Phase 5 反算） |
| `CompressionPlan.raster_budget` | `UserPreferences.target_size_mb` − `PreprocessResult.total_fixed_bytes` |
| `ImageResult.image_index` | `RasterPlan.image_index` ← `raster_images` 下标 |
| `PagePlan.skip` | `UserPreferences.per_page_overrides[].action == "skip"` |
