## 逐项说明

### ① 真实作品集 PDF 样本 —— 🔴 唯一会近期阻塞开发的事项

规则 6：测试不得用 AI 生成的假 PDF 蒙混。Phase 5（元素提取）验收需要：

| 文件 | 要求 | 用途 |
|------|------|------|
| `photo_heavy.pdf` | 渲染图/照片为主，≥10 页，最好 ≥30MB | 位图提取 + DPI 反算 + 分类 |
| `vector_heavy.pdf` | CAD/线稿/矢量图为主（如 Rhino/AutoCAD 导出） | 矢量路径 + 控制点统计 + 光栅化策略 |
| `mixed.pdf` | 图文混排的典型作品集 | 综合分类 + 端到端 |

- 放置位置：`backend/tests/fixtures/portfolios/`（`*.pdf` 已在 .gitignore，不会被提交）
- 来源建议：你自己/同学的作品集（取得同意）、你过往的课程排版文件
- 隐私提示：这些文件只在本机测试用，永不上传；含敏感个人信息的页面可先手动删掉
- 无免费替代：这是"真实性"本身，无法替代

### ② GitHub 私有仓库 —— 🟠

- 建 private repo（名字如 `portfolio-pdf-compressor`），把 remote URL 给我，我来 push
- 价值：代码异地备份；Phase 15 的 CI/CD 建议方案（GitHub Actions → ECR → EC2）依赖它
- 免费替代：无需替代，私有仓库免费

### ③ PDF 阅读器目视验收 —— 🟠

- 手册 Phase 10 要求：最终 PDF 用 Adobe Reader（免费）目视验证（Windows 上浏览器/Edge 也可作初检，但 Adobe 对书签/超链接/表单的还原最标准）
- 你要亲眼确认：页面顺序、图片位置、文字清晰度、书签/超链接可点

### ④ 扩充样本库 —— 🟡（Phase 13 前）

Phase 13 端到端矩阵需要：
- `text_heavy.pdf`（文字为主，~10MB）、`huge.pdf`（~200MB 超大件）补齐 5 类
- 另备 **20 份**不同风格作品集用于收敛成功率统计（目标 >90%）
- 建议现在开始顺手收集，收满即止；找同学要是最快路径

### ⑤ AWS 账号与 EC2 —— 🟢（Phase 15 前 1-2 天）

需要你亲手做（涉及支付方式和 root 凭证，AI 不应代办）：
1. 注册 AWS 账号（绑卡）
2. 创建 IAM 用户（AdministratorAccess 或按需最小权限），**不要用 root 日常操作**
3. 启动 EC2：区域选离目标用户近的（如 us-west-2 / ap-southeast-1）
4. 安全组：22（仅你的 IP）、80/443（0.0.0.0/0）
5. 分配弹性 IP 并绑定实例

费用估算（us-west-2 按需价，约）：

| 配置 | 月费用 | 说明 |
|------|--------|------|
| t3.large（2vCPU/8GB）+ 100GB gp3 | ~$61 + ~$8 ≈ **$69/月（~¥500）** | 手册推荐起步配置 |
| t3.medium（2vCPU/4GB）+ 50GB gp3 | ~$30 + ~$4 ≈ **$34/月（~¥245）** | 降配替代：能跑，200MB 大件会慢/紧张 |
| t2.micro/t3.micro 免费层（12 个月） | **$0** | 仅够演示小 PDF；100MB+ 会内存不足，不能作为正式方案 |

省钱选项：先买 1 个月按需验证，稳定后转 1 年期 Savings Plan（约 6-7 折）；不用时 Stop 实例（EBS 保留，计算停止计费）。

### ⑥ 域名 —— 🟢（注意 DNS 生效提前量）

- 任意注册商（Namecheap/Cloudflare/阿里云）；`.com` ~¥70-110/年
- 需要两条解析：`api.你的域名` → EC2 弹性 IP（A 记录）；`www/@` → Vercel（CNAME，Vercel 会给指引）
- Let's Encrypt 证书签发依赖域名解析已生效，所以最晚 Phase 15 开始前 2-3 天完成
- 免费替代：`nip.io` 魔法域名（如 `1.2.3.4.nip.io`）可拿到证书做演示，但不适合对外发布

### ⑦⑧ Vercel / 邮箱 —— 🟢

- Vercel：GitHub 账号直接 OAuth 注册，import 仓库的 `frontend/` 目录即可；Hobby 免费层完全够用（前端只是静态 + API 转发）
- 邮箱：Let's Encrypt 注册与 CloudWatch 告警接收，用你现有邮箱即可

### ⑨ S3（可选）—— ⚪

- 仅当你想要"长期归档"功能才需要；核心流程用 EC2 本地 EBS 即可
- 若启用：建桶 + 给 EC2 的 IAM Role 加读写权限，把桶名经环境变量 `AWS__S3_BUCKET` 注入

## 并行时间线估计（"最快多久能上线"）

假设：每个 Phase 完成后你当天内批准；外部事项按上表并行推进；样本 PDF 明天到位。

```
第 1 周   Phase 4 拆页 → Phase 5 提取（🔴样本到位） → Phase 6 分类 → Phase 7 预处理
第 2 周   Phase 8 决策引擎（专项 review）→ Phase 9 执行+收敛 → Phase 10 重组（🟠目视验收）
第 3 周   Phase 11 任务编排 → Phase 12 API → Phase 13 测试矩阵（🟡20 份样本到位）
第 4 周   Phase 14 前端 → Phase 15 部署（🟢AWS/域名/Vercel 已就绪）→ 公网冒烟
```

- **乐观（审批零等待、样本即时到位、无大返工）：约 3 周上线**
- **现实（审批隔天、Phase 8/9 有一轮返工、部署调试 2 天）：约 4-5 周上线**
- 关键路径：样本 PDF（影响第 1 周）→ Phase 8/9 收敛调试（第 2 周弹性最大）→ DNS/证书（第 4 周）
- 最容易白等的坑：DNS 生效 + 证书签发，所以⑥刻意要求提前 2-3 天
